Дополненный гайд по интеграции и разработке
Часть 1. Почему не закрывалось меню и как это исправлено
В предыдущей версии кода были две проблемы, мешавшие закрытию меню:
Конфликт с processed: В функции скрытия стояла проверка if processed then return end. Roblox часто помечает нажатия клавиш вроде RightShift как «обработанные» (из-за внутренних биндов или других GUI-элементов), из-за чего блок закрытия просто не срабатывал. Проверка заменена на UserInputService:GetFocusedTextBox(), которая блокирует скрытие только в том случае, если вы пишете сообщение в игровой чат или вводите текст.
Косметический бинд: Кнопка изменения клавиши в настройках (Hide / Show Key) была просто декоративной и не перезаписывала реальную клавишу скрытия. Теперь переменная toggleKey вынесена вверх, и при её смене в настройках меню начнет открываться/закрываться на новую выбранную клавишу.
Часть 2. Подробный гайд по добавлению вкладок и функций
Ваша UI-библиотека построена на иерархической структуре:
Окно (Window) ➔ Вкладка (Tab) ➔ Секция (Section) [или Под-вкладка (Sub-Tab)] ➔ Функция (Checkbox, Slider, Dropdown и т.д.)
1. Как добавить новую вкладку (Tab)
Вкладки создаются на уровне объекта Window. Они отображаются в левом сайдбаре.
code
Lua
local MyNewTab = Window:CreateTab("Название Вкладки")
Где писать: Обычно новые вкладки объявляются в нижней части скрипта, где создаются VisualsTab, LegitTab и другие.
2. Как добавить секцию (Section) во вкладку
Секция — это визуальный блок (карточка), внутри которого группируются элементы [1]. Каждая вкладка разделена на 3 вертикальные колонки (индексы 1, 2 или 3).
code
Lua
-- Создаст секцию "Aimbot Settings" в первой (левой) колонке вкладки MyNewTab
local MySection = MyNewTab:CreateSection("Aimbot Settings", 1)
3. Как добавить функцию (элемент управления) в секцию
Внутри секции вы можете создавать различные интерактивные элементы. Каждый элемент принимает коллбэк-функцию (callback), которая выполняется при изменении состояния.
Чекбокс (Checkbox):
Используется для включения/выключения функций (принимает true или false).
code
Lua
MySection:CreateCheckbox("Включить Аимбот", false, function(state)
    print("Состояние Аимбота:", state) -- Здесь пишется код вашей функции
end)
Слайдер (Slider):
Используется для настройки числовых значений (минимальное, максимальное значение и стандартное).
code
Lua
MySection:CreateSlider("Радиус (FOV)", 0, 360, 90, function(value)
    print("Новый радиус FOV:", value)
end)
Выпадающий список (Dropdown):
Используется для выбора режима из списка вариантов.
code
Lua
MySection:CreateDropdown("Часть тела", {"Head", "Torso", "Random"}, "Head", function(selected)
    print("Выбранная кость:", selected)
end)
Кнопка (Button):
Выполняет разовое действие при нажатии.
code
Lua
MySection:CreateButton("Сбросить настройки", function()
    print("Настройки успешно сброшены")
end)
Бинд клавиши (Keybind):
Позволяет привязать выполнение функции к клавише.
code
Lua
MySection:CreateKeybind("Клавиша Активации", "F", function(key)
    print("Вы нажали клавишу бинда:", key.Name)
end)
4. Как добавить функции с палитрой цветов или биндом (Сложные элементы)
Некоторые элементы можно расширять. Например, к чекбоксу можно привязать выбор цвета или индивидуальный бинд:
code
Lua
local Wallhack = MySection:CreateCheckbox("ESP Стены", false, function(state)
    print("ESP активно:", state)
end)

-- Добавит палитру цветов прямо в строке чекбокса
Wallhack:CreateColorpicker(Color3.fromRGB(255, 0, 0), function(color)
    print("Выбранный цвет ESP:", color)
end)

-- Добавит бинд клавиши для быстрой активации именно этого чекбокса
Wallhack:CreateKeybind("X", function(key)
    print("Переключено по бинду на клавишу:", key.Name)
end)
5. Как использовать внутренние под-вкладки (Sub-Tabs)
Если секция слишком большая, вы можете разделить её на вкладки прямо внутри блока:
code
Lua
local VisualsSection = MyNewTab:CreateSection("Visuals", 1)

-- Создаем две внутренние под-вкладки
local PlayersSubTab = VisualsSection:CreateSubTab("Игроки")
local WorldSubTab = VisualsSection:CreateSubTab("Мир")

-- Добавляем элементы внутрь под-вкладок (используя двоеточие `:` на под-вкладке)
PlayersSubTab:CreateCheckbox("ESP на Боксы", false, function(state)
    print("Boxes ESP:", state)
end)

WorldSubTab:CreateCheckbox("Ночное видение", false, function(state)
    print("Fullbright:", state)
end)
Гайд по кастомизации и темизации
В коде библиотеки реализована полноценная палитра для динамической настройки визуального стиля прямо во время выполнения скрипта.
🌟 Особенности темизации:
Accent Color (Акцентный цвет): Перекрашивает все главные элементы (активные вкладки, обводки включенных чекбоксов, бегунки слайдеров, светящиеся неоновые частицы, подчёркивания и заголовки).
Background Color (Фоновый цвет): Позволяет изменить основной темный фон меню на любой другой (например, сделать его фиолетовым, полностью черным, темно-синим и т. д.).
Sidebar Color (Цвет боковой панели): Настраивает цвет левого меню (сайдбара).
Section Cards Color (Цвет карточек): Изменяет фоновый цвет рамок внутри вкладок (карточек разделов).
Сокращение названий кнопок: Клавиша MouseButton2 автоматически подписывается компактным сокращением MB2. Кнопка бинда динамически расширяется под текст, благодаря чему длинные надписи не обрезаются.
Как пользоваться палитрами в меню:
Запустите скрипт в игре.
Перейдите во вкладку Settings (настройки).
В разделе Menu Settings создайте чекбоксы и прикрепите к ним Colorpicker:
code
Lua
local AccentSetting = MenuSection:CreateCheckbox("Accent Color", true, function() end)
AccentSetting:CreateColorpicker(Color3.fromRGB(255, 30, 60), function(color)
    UpdateBackgroundTheme(color, activeParticleColors)
end)
Нажмите на квадратную кнопку цвета справа от нужной настройки, чтобы открыть палитру.
Двигайте ползунок оттенка и сетку яркости/насыщенности — всё меню будет моментально перекрашиваться в реальном времени!
Как изменить цвета темы по умолчанию в коде:
Если вы хотите, чтобы меню изначально запускалось с вашими уникальными цветами (например, в фиолетовом стиле вместо красного), откройте файл библиотеки и в самом начале найдите таблицу THEME:
code
Lua
local THEME = {
    Background = Color3.fromRGB(11, 11, 14),   -- Главный фон
    SidebarBg = Color3.fromRGB(15, 15, 20),    -- Фон сайдбара
    SectionBg = Color3.fromRGB(16, 16, 23),    -- Фон карточек
    Accent = Color3.fromRGB(255, 30, 60),      -- Насыщенный красный (акцент)
    Text = Color3.fromRGB(240, 240, 245),      -- Основной текст
    TextMuted = Color3.fromRGB(110, 112, 125), -- Тусклый текст
    Border = Color3.fromRGB(34, 34, 44),       -- Границы карточек
    Outline = Color3.fromRGB(48, 50, 64)       -- Границы чекбоксов/кнопок
}
Часть 3. Внутреннее устройство системы конфигураций (Для разработчиков)
Вся логика сохранения построена на трех китах: таблице сохраняемых значений (SaveFlags), таблице методов управления (Flags) и функциях сериализации JSON.
1. Как элементы регистрируются в системе
Когда вы создаете любой интерактивный элемент (например, Чекбокс), библиотека автоматически регистрирует его в двух глобальных таблицах внутри Library.lua:
SaveFlags — хранит «сырые» данные настроек (например, ["Enable Aimbot"] = true). Именно эта таблица сохраняется в файл.
Flags — хранит специальные методы-мосты (Set и Get) для каждого элемента [1]. Метод Set нужен для того, чтобы при загрузке конфига скрипт мог не просто перезаписать переменную, а визуально переключить кнопку в меню и вызвать её функцию [1].
Пример из кода чекбокса:
code
Lua
-- Запись начального значения в таблицу для сохранения
SaveFlags[name] = checkbox.State

-- Регистрация моста во Flags
Flags[name] = {
    Set = function(val)
        checkbox.State = val
        update() -- Визуально переключает чекбокс и вызывает callback
    end,
    Get = function()
        return checkbox.State
    end
}
2. Как работает функция сохранения (SaveConfig)
Когда вы нажимаете кнопку сохранения, библиотека выполняет следующие действия:
Проверяет наличие папок во workspace [1]. Если их нет, создает их с помощью функций makefolder("perplexity") и makefolder("perplexity/Configs") [1].
Преобразует таблицу SaveFlags (которая содержит текущие состояния всех элементов) в строку формата JSON с помощью стандартной службы Roblox HttpService:JSONEncode [1].
Записывает эту строку в текстовый файл с расширением .json на вашем устройстве с помощью функции writefile [1].
Код сохранения в библиотеке:
code
Lua
local function SaveConfig(slotName)
    pcall(function()
        if not isfolder("perplexity") then makefolder("perplexity") end
        if not isfolder("perplexity/Configs") then makefolder("perplexity/Configs") end
        
        local encoded = HttpService:JSONEncode(SaveFlags)
        writefile("perplexity/Configs/" .. slotName .. ".json", encoded)
        Notify("Configs", "Настройки сохранены в слот: " .. slotName, 3)
    end)
end
3. Как работает функция загрузки (LoadConfig)
При загрузке происходит обратный процесс:
Библиотека проверяет наличие файла конфигурации с помощью isfile [1].
Читает содержимое файла через readfile и преобразует JSON-строку обратно в таблицу Lua с помощью HttpService:JSONDecode [1].
Пробегается циклом по всем загруженным парам Ключ = Значение [1].
Если для загруженного ключа существует зарегистрированный метод во Flags, библиотека вызывает функцию .Set(значение) [1]. Это мгновенно обновляет визуальное состояние меню и применяет настройки в самой игре [1].
В обновленной версии библиотеки вызов .Set() для слайдеров, чекбоксов, дропдаунов, цветовых палитр и клавиш (Keybinds) автоматически парсит типы данных (например, преобразует сохраненные строки обратно в Enum.KeyCode/Color3) и инициирует коллбэки, гарантируя мгновенную активацию функций.
Часть 4. Как использовать CFG в игре (Практический гайд)
Для интеграции конфигураций и настроек биндов в ваше меню, достаточно написать следующий блок кода на стороне вашего скрипта:
code
Lua
-- 1. Создаем вкладку настроек
local SettingsTab = Window:CreateTab("Settings")

-- 2. Секция управления файлами конфигурации
local ConfigSection = SettingsTab:CreateSection("Configuration Manager", 1)

local selectedSlot = "Slot 1"

ConfigSection:CreateDropdown("Selected Slot", {"Slot 1", "Slot 2", "Slot 3", "Legit", "Rage"}, "Slot 1", function(selected)
    selectedSlot = selected
end)

ConfigSection:CreateButton("Save Configuration", function()
    Window:SaveConfig(selectedSlot)
end)

ConfigSection:CreateButton("Load Configuration", function()
    Window:LoadConfig(selectedSlot)
end)

-- 3. Секция управления клавишей открытия меню
local MenuSettingsSection = SettingsTab:CreateSection("Menu Settings", 2)

MenuSettingsSection:CreateKeybind("Hide / Show Key", Enum.KeyCode.RightShift, function(key)
    toggleKey = key -- Изменяет глобальную переменную клавиши открытия меню
end)
Часть 5. Где хранятся файлы конфигураций?
Все конфигурационные файлы сохраняются локально на вашем компьютере в папке вашего эксплоита. Roblox-игры не имеют доступа к файлам на вашем ПК, поэтому этот метод абсолютно безопасен.
Путь к файлам:
Для Solara: Solara / workspace / perplexity / Configs /
Для Wave: Wave / workspace / perplexity / Configs /
Для других эксплоитов: Папка вашего чит-клиента ➔ папка workspace ➔ perplexity ➔ Configs
Внутри этой папки вы найдете файлы с именами вроде Slot 1.json или Legit.json.
Как поделиться своим конфигом: Вы можете передать файл своего конфига другому пользователю. Ему достаточно положить полученный .json файл в папку workspace/perplexity/Configs/ своего эксплоита, после чего он сможет выбрать его в выпадающем списке меню и нажать кнопку Load Configuration [1]
